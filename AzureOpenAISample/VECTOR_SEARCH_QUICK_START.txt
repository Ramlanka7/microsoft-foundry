Q: What is vector search?
A: Converts text to embeddings (vectors), finds similar content using
   cosine similarity instead of keyword matching.

Q: Why use vector search?
A: Understands meaning not just words. Finds "car" and "automobile"
   as similar even though keywords differ.

Q: What is HNSW?
A: Hierarchical Navigable Small World - ANN algorithm that builds
   multi-layer graph for fast (log time) vector search.

Q: Vector vs Traditional search?
A: Traditional (BM25) = keyword matching, fast, misses synonyms
   Vector = semantic similarity, slower, understands meaning
   Hybrid = BEST (combines both with RRF)

Q: What is RRF?
A: Reciprocal Rank Fusion - merges multiple result sets by rank
   position, not score. Used in hybrid search.

Q: Cosine similarity?
A: Measures angle between vectors. Range -1 to 1. 
   1=identical, 0=unrelated. Formula: (AÂ·B)/(||A||Ã—||B||)

Q: Embedding dimensions?
A: Determined by model. ada-002 = 1536, must match index config.

Q: Vector vs Fine-tuning?
A: Vector (RAG) = external knowledge, updatable, transparent
   Fine-tuning = behavior/style, static, black box
   Often use BOTH!
